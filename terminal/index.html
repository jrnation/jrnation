How to Find Text Inside Files in Linux (`grep`)

You don't need to open a file to read it. The grep command acts like a search engine for your terminal, allowing you to instantly scan massive log files or code directories for specific words.

1. The Basic Search

To find a specific word inside a single file, type grep, the word in quotes, and the file name.

grep "error" syslog.txt

2. Ignore Case Sensitivity (-i)

By default, grep is strictly case-sensitive. It treats "Error" and "error" as two completely different words. To find both, add the -i flag.

grep -i "failed" syslog.txt

3. Count the Matches (-c)

If you don't want to see the actual lines, but just want to know how many times an error occurred, use the count flag.

grep -c "error" syslog.txt
