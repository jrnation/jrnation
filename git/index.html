Git: Hell I Know

Welcome to the nightmare. Every developer pretends to understand Git, but most are just copy-pasting commands from StackOverflow and praying they don't delete the company's entire codebase.

Git is a Version Control System. It acts as a time machine for your folders, taking permanent snapshots of your code so you can safely undo catastrophic mistakes.

The Three Zones

To survive Git, you must understand that your files exist in one of three physical states:

  • 1. The Workspace (Working Directory): Your chaotic desk. These are the files you are currently typing code into. Git sees them, but it isn't saving them yet.
  • 2. The Staging Area (The Loading Dock): The waiting room. You explicitly place files here when you are done editing them and want to include them in the next save state.
  • 3. The Repository (The Vault): The permanent timeline. Once files move here, they are locked into an indestructible snapshot (a commit).

🔥 The Simulation

We built a visual dashboard on the right. Watch the boxes as you type the commands below into the terminal.

  1. Type git init to wake the engine up and spawn some files on your desk.
  2. Type git status to see what the radar detects.
  3. Type git add index.html. Watch the file physically jump to the Staging Area!
  4. Type git add . to sweep the rest of the files into Staging.
  5. Type git commit -m "My first save" to lock them into the Vault.
