Window Managers: Ditch the Mouse

Switching to a WM is like Rock Lee dropping his training weights. | 👤 By JR Nation

Right now, you are probably using a Desktop Environment (DE) like GNOME, KDE Plasma, or Cinnamon. A DE is a complete, monolithic package. Out of the box, it provides a start menu, a taskbar, a Bluetooth manager, a settings app, a file browser, and a lock screen. It is comfortable, incredibly heavy, and assumes you want to use your computer like a normal person.

But you are not a normal person anymore. You are a Power User. You know the terminal. You know your scripts. You know that lifting your hand off the keyboard to grab a mouse slows you down. It is time to strip away the corporate bloat, take absolute control of your pixels, and install a Window Manager (WM).

1. The Great Schism (Compositors vs. WMs)

A pure Window Manager does exactly one thing: it calculates the math required to draw window borders on your screen. That is it.

If you install a pure WM, strip out your DE, and log in, there is no start menu. There is no WiFi icon. There is no wallpaper. You just get a pitch-black screen. If you want a wallpaper, you have to run a background daemon to draw one. If you want a taskbar, you have to install a separate application entirely and configure it in plain text.

Why would anyone willingly subject themselves to this? Unrivaled Performance.

  • GNOME uses about 1.5 GB of RAM just sitting idle on your desktop to power all its heavy background services.
  • A custom Window Manager can use as little as 150 MB of RAM, freeing up massive resources for compiling code, running virtual machines, or rendering video.

2. Tiling Philosophies (Manual vs. Dynamic)

Windows and macOS use Stacking (or Floating) window management. You drag a window with your mouse, it floats on the screen, and you stack other windows on top of it. This creates a chaotic desk where you spend half your time resizing boxes trying to find the terminal hidden underneath your browser.

Power users use Tiling. In a Tiling Window Manager (TWM), windows never overlap. When you open a terminal, it mathematically claims 100% of the screen. When you open a web browser, the screen splits perfectly 50/50. But how they split depends on the philosophy of the WM:

  • Manual Tiling: (Used by i3 and Sway). You explicitly tell the WM where the next window should go before you open it. You press a shortcut to split the screen horizontally, then open your app. You have absolute control over the layout.
  • Dynamic Tiling: (Used by AwesomeWM, dwm, and Hyprland). The WM uses complex algorithms (like Binary Space Partitioning or Fibonacci spirals) to automatically place windows for you based on a master/stack layout. The largest window is the "Master", and new windows automatically stack in a smaller column next to it.

3. The Modular Ecosystem (Waybar, Rofi, Dunst)

Because a Window Manager doesn't come with a taskbar or a start menu, you get to build your own Desktop Environment using highly specialized, independent modular programs. This is where "Ricing" reaches its peak form.

  • The Bar: Instead of the heavy Windows taskbar, users install tools like Polybar (for X11) or Waybar (for Wayland). You write a script to tell the bar exactly what to display—from your CPU temperature, to the current song playing on Spotify, to your custom workspaces.
  • The Launcher: Because there is no start menu, you install an application launcher like Rofi or Wofi. Pressing a shortcut spawns a lightning-fast, searchable text menu in the center of your screen. Type the first three letters of an app, hit enter, and it launches instantly.
  • The Notifications: Window managers don't know how to pop up a notification when you get a Discord message. You must run a lightweight notification daemon like Dunst or Mako to draw the pop-ups.

4. The Modern Titans: i3, Sway, Hyprland

If you want to ascend to the TWM lifestyle, these are the modern heavyweights you must choose between:

  • i3wm: The absolute classic. It runs on the legacy X11 display server. It is rock-solid, incredibly well documented, uses almost zero system resources, and is the absolute best place for a beginner to start learning how manual tiling works.
  • Sway: A literal drop-in replacement for i3, but built for the modern, highly secure Wayland display server protocol. It even reads your old i3 configuration file perfectly. If you have an AMD or Intel GPU, this is the gold standard for tearing-free stability.
  • Hyprland: The current undisputed king of the r/unixporn subreddit. Built in C++ around the wlroots Wayland compositor, it brings fluid, buttery-smooth animations, blurred transparent windows, and gorgeous eye-candy to the tiling world. It requires a bit more GPU horsepower but looks absolutely incredible.

5. The "Mod" Key Workflow

In a TWM, you almost never touch your mouse. Everything is controlled by a designated "Modifier" key (usually the Windows/Super key or the Alt key) paired with a letter.

Example TWM Shortcuts (Usually customizable in ~/.config):

Mod + Enter — Spawns a new Terminal instantly.

Mod + Shift + Q — Kills the currently focused window.

Mod + Arrow Keys — Shifts your focus physically between the split tiles.

Mod + 2 — Instantly throws you to virtual Workspace #2.

Mod + Shift + E — Quits from the Window Manager and logs you out to the Display Manager.

⚠️ THE TRAP: Before you log into a new Window Manager, you MUST know the default keyboard shortcut to open a terminal (usually Mod+Enter). Because there are no graphical menus, if you log in and don't know the shortcut to open a terminal to edit your config file, you will be permanently trapped staring at a black screen until you force reboot your PC with the physical power button.

Mastering a tiling window manager turns your keyboard into an extension of your brain. Once you experience the speed of tiling, you will never want to use a floating, clicking, bloated desktop environment ever again.
