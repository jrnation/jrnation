GRUB: Pimp My Bootloader

Because staring at a generic text menu every morning is depressing. | 👤 By JR Nation

The Grand Unified Bootloader (GRUB) is the very first piece of software you interact with when you turn on a dual-boot PC. It is the gatekeeper that asks you which operating system or kernel you want to load. Out of the box, however, it is incredibly ugly—usually just generic white text floating on a pitch-black background.

But GRUB is a highly modular piece of software. You can change the countdown timer, set a default OS so you don't have to babysit the boot sequence, pass secret commands directly to the Linux kernel, and even add a custom background image so your system looks beautiful before the operating system even loads.

1. The Master File (/etc/default/grub)

Here is a common beginner mistake: new users will try to edit the actual GRUB script located at /boot/grub/grub.cfg. Never do this. That file is automatically generated by the system, and any manual edits you make there will be instantly overwritten and destroyed the next time your kernel updates.

To customize GRUB, you must edit the central configuration template. From this template, the system will generate the actual code. Open it using your terminal text editor:

$ sudo nano /etc/default/grub

2. Changing the Timeout & Default OS

Once you are inside the file with nano, look for these specific configuration lines:

  • GRUB_TIMEOUT=10: This dictates exactly how many seconds the menu stays on the screen before auto-selecting an OS. Change it to 3 or 5 to speed up your boot time. Change it to -1 if you want it to wait forever until you explicitly press a key on your keyboard.
  • GRUB_DEFAULT=0: This tells GRUB which list item to boot automatically when the timer runs out. 0 is the first item on the list (usually Linux). 1 is the second item.
  • The Memory Trick: If you want GRUB to automatically remember the last OS you booted and highlight it for next time, change the default line to GRUB_DEFAULT=saved and add a brand new line directly below it that says GRUB_SAVEDEFAULT=true.

3. Pro-Tier: Kernel Parameters (quiet splash)

Inside the config file, you will see a line that looks like this: GRUB_CMDLINE_LINUX_DEFAULT="quiet splash".

This is where you pass commands directly to the Linux Kernel before it boots. By default, distributions like Ubuntu and Mint include the words quiet (which tells the kernel to hide its complex text logs) and splash (which tells the system to display a cute, graphical loading screen with a spinning logo).

If you want your computer to look like a Hollywood hacker movie when it boots up, simply delete those two words so the line reads: GRUB_CMDLINE_LINUX_DEFAULT="". Now, when you turn on your PC, you will see the raw "Matrix" code of Systemd loading every single service and driver in real-time. This isn't just for looks; it is incredibly useful for troubleshooting exactly where a boot process is getting stuck.

4. Adding a Custom Background Image

To put a cool image behind the text menu, you just need to point GRUB to a picture on your hard drive. (Note: the image must be a .png, .jpg, or .tga, and it works best if it exactly matches your monitor's native resolution).

Add this line to the very bottom of the /etc/default/grub file:

GRUB_BACKGROUND="/usr/share/backgrounds/my_cool_wallpaper.png"

# Make sure you provide the exact, absolute path to where your image is saved.

Press Ctrl + O to save, hit Enter to confirm, and Ctrl + X to exit Nano.

5. The Crucial Step: Compiling GRUB

You saved the text file, but if you reboot right now, absolutely nothing will change. You have to tell the system to read that template file and compile it into the actual bootloader script.

⚠️ Run the Update Command:

If you are on Ubuntu, Debian, or Mint:

sudo update-grub

If you are on Arch, Endeavour, or Fedora:

sudo grub-mkconfig -o /boot/grub/grub.cfg

You will see the terminal output text confirming it found your custom background image and your operating systems. Reboot your PC, and enjoy your customized boot sequence.
