GRUB: Pimp My Bootloader

Because staring at a black and white text menu every morning is depressing. | 👤 By JR Nation

The Grand Unified Bootloader (GRUB) is the first piece of software you see when you turn on a dual-boot PC. Out of the box, it is incredibly ugly. It’s just generic white text on a pitch-black background.

But GRUB is highly customizable. You can change the countdown timer, set a default OS so you don't have to babysit the boot sequence, and even add a custom background image so your system looks beautiful before the operating system even loads.

1. The Master File (/etc/default/grub)

You do not edit the actual GRUB menu directly. Instead, you edit a configuration file, and then tell Linux to generate a new menu based on your rules. The file you need to edit is located here:

$ sudo nano /etc/default/grub

2. Changing the Timeout & Default OS

Once you are inside the file with nano, look for these specific lines:

  • GRUB_TIMEOUT=10: This dictates how many seconds the menu stays on the screen before auto-selecting an OS. Change it to 3 or 5 to speed up your boot time. Change it to -1 if you want it to wait forever until you explicitly press a key.
  • GRUB_DEFAULT=0: This tells GRUB which list item to boot automatically. 0 is the first item on the list (usually Linux). 1 is the second item. If you want it to automatically remember the last OS you booted, change this to GRUB_DEFAULT=saved and add a new line below it that says GRUB_SAVEDEFAULT=true.

3. Adding a Custom Background Image

To put a cool image behind the text, you just need to point GRUB to a picture on your hard drive. (Note: the image should be a .png or .jpg, and it helps if it matches your monitor's native resolution).

Add this line to the very bottom of the /etc/default/grub file:

GRUB_BACKGROUND="/usr/share/backgrounds/my_cool_wallpaper.png"

# Make sure you provide the exact, absolute path to where your image is saved.

Press Ctrl + O to save, hit Enter to confirm, and Ctrl + X to exit Nano.

4. The Crucial Step: Updating GRUB

You saved the text file, but if you reboot right now, absolutely nothing will change. You have to tell the system to compile that text file into the actual bootloader script.

⚠️ Run the Update Command:

If you are on Ubuntu, Debian, or Mint:

sudo update-grub

If you are on Arch, Endeavour, or Fedora:

sudo grub-mkconfig -o /boot/grub/grub.cfg

You will see the terminal output text confirming it found your background image and your operating systems. Reboot your PC, and enjoy your customized boot sequence.
