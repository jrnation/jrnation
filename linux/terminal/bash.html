Bash Scripting: The Art of Automation

Why do it yourself when the machine can do it for you? | 👤 By JR Nation

You now know how to navigate the filesystem, install software, and use text editors. But typing out long strings of commands every single day gets tedious. What if you could stack all your favorite commands into a single text file and execute them all at once with a single word?

That is what Bash Scripting is. You are writing a recipe for the terminal to follow. It is the ultimate superpower of the Linux ecosystem, allowing you to automate literally any task on your machine.

💡 Practice Automation: Writing scripts can feel abstract until you actually run one. Open the JR Nation Terminal Playground, create a file, and practice writing your first script safely.

1. What is Bash?

When you type commands into your terminal, you are talking to a "Shell"—a program that takes your text, interprets it, and passes it to the kernel. On most Linux distributions, the default shell is called Bash (Bourne Again SHell).

A Bash script is literally just a plain text file containing a list of terminal commands from top to bottom. The computer reads line 1, executes it, then moves to line 2, just like reading a book.

2. The Magic Words: The Shebang

How does the computer know a text file is actually an executable script and not just a grocery list? You have to put a special code on the very first line of the file. In the Linux world, this is called the Shebang.

#!/bin/bash

# This tells the system to use the Bash interpreter located in the /bin folder.

3. Project: The Auto-Updater

Let's use the nano text editor you learned about in the last chapter to write a script that updates your entire system automatically.

Open your terminal and create a new file: nano update_my_pc.sh (The .sh extension stands for shell script).

Type the following code exactly as you see it:

#!/bin/bash


echo "Starting system update sequence..."

# The 'echo' command just prints text to the screen so you know what is happening.


sudo apt update && sudo apt upgrade -y

# The -y flag automatically answers 'yes' to any prompts.


sudo apt autoremove -y

# Cleans up old, unnecessary dependencies.


echo "Update complete! Your system is secure."

Press Ctrl + O to save, Enter to confirm, and Ctrl + X to exit Nano.

4. Giving it Life (chmod +x)

You have written the script, but if you try to run it right now, Linux will aggressively block you. As we learned in the Filesystem & Permissions guide, Linux protects you from executing text files by default.

You need to hand over the key. Run this command to make the file executable:

$ chmod +x update_my_pc.sh

Now, to unleash the script, simply type a dot and a slash (which means "execute this file in my current directory") followed by the file name:

$ ./update_my_pc.sh

Sit back and watch the matrix flow. You just programmed your operating system.

💡 The Next Level: Bash isn't just a list of commands. It's a full programming language. You can add if/else statements, variables, and loops. If you want a script to backup your files every night while you sleep automatically, look into a tool called Cron jobs.
