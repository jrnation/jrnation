User Customization: Dressing the Avatar

You gave them an account. Now give them an identity. | 👤 By JR Nation

Creating a user account with the right permissions is the hard part. But logging into a blank grey silhouette on the lock screen is boring. Once an account exists, you can customize how that user is presented to the system, from their profile picture to the very shell they use to type commands.

Table of Contents

  • 1. The Lock Screen Avatar (Profile Pic)
  • 2. Changing the Display Name
  • 3. Upgrading the Default Shell (chsh)

1. The Lock Screen Avatar (Profile Pic)

When you boot up your PC, the Display Manager (the graphical login screen) looks for an image to associate with your account. There are two ways to change this.

The Easy Way (GUI)

If you are using a standard Desktop Environment like GNOME or KDE Plasma, this is built right into the settings.

  • Open your system Settings app.
  • Navigate to the Users or Accounts tab.
  • Click on the blank avatar next to your name and select an image file from your computer.

The Hacker Way (The .face trick)

If you are on a minimal setup, using a Window Manager, or just want to do it via the terminal, you can use a classic Linux trick. Most login managers automatically look for a hidden image file named .face right inside your home directory.

$ cp /path/to/my_cool_picture.png ~/.face

# Copies your picture into your home folder and hides it under the name '.face'

# Make sure the image is a square (e.g., 256x256 pixels) for best results.


$ cp /home/jaya/Pictures/my_cool_picture.png ~/.face

# Example: If your picture is in your Pictures folder, this copies it correctly.

Restart your computer, and the login manager will automatically grab that hidden image and use it as your avatar.

2. Changing the Display Name

Your username (what you type into the terminal to log in) cannot have spaces or capital letters. But your display name (what shows up on the lock screen and in the start menu) can be whatever you want.

To change a user's display name without altering their actual system username, use the usermod command with the -c (Comment/GECOS) flag.

$ sudo usermod -c "Jaya The Admin" jaya

# The login screen will now say "Jaya The Admin", but the terminal user remains 'jaya'.

3. Upgrading the Default Shell (chsh)

By default, when a user opens the terminal, they are talking to the Bash shell. However, many power users prefer to use a modernized shell like Zsh or Fish, which offer incredible features like auto-suggestions, syntax highlighting, and deeper visual customization.

If you install a new shell (e.g., sudo apt install zsh), you need to tell Linux to make it your default account shell. You do this with the chsh (Change Shell) command.

$ chsh -s $(which zsh)

# -s sets the shell. $(which zsh) automatically finds where the Zsh program is located.


$ chsh -s $(which fish)

# Example: If you prefer Fish, this command will set it as your default shell.

💡 Final Step: For the shell changes to take effect, you must completely log out of your user account and log back in (or simply reboot your computer). The next time you open the terminal, you will be greeted by your new environment!
