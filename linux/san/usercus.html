User Customization: Dressing the Avatar

You gave them an account. Now give them an identity. | 👤 By JR Nation

Creating a user account with the right permissions is just the mechanical part of the job. But logging into a blank grey silhouette on the lock screen every day is a miserable experience. Once a secure account exists, you can fundamentally alter how that user interacts with the system.

Because Linux respects the boundary between the Root system and the User space, every single account on a machine can have a completely isolated experience. User A can have a heavily animated, futuristic terminal shell, while User B operates in a strict, minimalist environment. Customizing these profiles isn't just about aesthetics; it is about building a workflow that makes you faster and more lethal on the command line.

1. The Lock Screen Avatar (Display Managers)

When you boot up your PC, the graphical login screen you see is controlled by a piece of software called a Display Manager (common ones include GDM3 for GNOME, SDDM for KDE Plasma, and LightDM for XFCE). This software looks into your user folder to see if you have claimed an identity.

The Standard Way (GUI)

If you are using a standard, heavy Desktop Environment, the developers have built graphical hooks for this process.

  • Open your system Settings application.
  • Navigate to the Users or Accounts tab.
  • Unlock the settings using your sudo password, click on the blank avatar next to your name, and select a .png or .jpg file from your hard drive.

The Hacker Way (The .face trick)

If you are running a hyper-minimalist Window Manager, or managing a server entirely through the command line, you won't have a graphical settings menu. Instead, you can exploit a classic Unix trick. Most Display Managers are hardcoded to look for a hidden image file named exactly .face or .face.icon sitting at the root of your home directory.

$ cp /home/jaya/Pictures/my_cool_avatar.png ~/.face

# Copies your picture into your home folder and hides it under the name '.face'


$ cp ~/.face ~/.face.icon

# Creates a duplicate to ensure compatibility with older Display Managers.

Pro-Tip: Make sure the image is perfectly square (e.g., 256x256 pixels) to prevent the login screen from squishing or stretching your logo unrecognizably.

2. Modifying the GECOS Field (Display Name)

Your actual system username—the one you type into the terminal to log in—must be strictly lowercase, with no spaces or special characters. But the display name (what shows up on the lock screen with a capital letter and spaces) can be whatever you want.

Historically, in the early days of Unix, this meta-information was stored in the GECOS field (General Electric Comprehensive Operating System), a text string inside the /etc/passwd file used to hold a user's full name, office room number, and phone number. Today, Linux still uses this exact same architecture.

To safely change a user's display name without risking corruption of the critical /etc/passwd file, you use the usermod command with the -c (Comment) flag.

$ sudo usermod -c "Jaya The Admin" jaya

# The login screen will now display "Jaya The Admin", but the underlying system username remains 'jaya'.

3. Workflow Hacks: Creating Bash Aliases

The most powerful user customization you can perform doesn't involve graphics at all; it involves creating shortcuts for your brain. If you find yourself typing the exact same long, annoying command ten times a day, you can create an Alias. An alias is a custom word that you define, which the terminal automatically expands into a full command.

These shortcuts are saved in a hidden configuration file in your home directory called .bashrc.

$ nano ~/.bashrc

# Scroll to the absolute bottom of the file and add your custom shortcuts:


alias update="sudo apt update && sudo apt upgrade -y"

alias yeet="sudo rm -rf"

alias ports="sudo ufw status verbose"

Save the file and restart your terminal. Now, instead of typing out the entire system update command, you literally just type update and press Enter. The system will handle the rest. This alone will save you hours of typing over a year.

4. Upgrading the Default Shell (Bash vs. Zsh vs. Fish)

By default, when any user opens the terminal on almost any Linux distribution, they are interacting with the Bash (Bourne Again SHell) interpreter. It is rock-solid, legendary, and heavily documented. But it is also old, and frankly, a bit ugly out of the box.

Modern power users often choose to swap their default shell to an upgraded alternative. The two main competitors are:

  • Zsh (Z Shell): The industry favorite for developers. When paired with a framework called Oh-My-Zsh, you get access to thousands of custom themes, git-branch tracking in your prompt, and powerful plugin ecosystems. Apple actually made Zsh the default shell for macOS because of its immense flexibility.
  • Fish (Friendly Interactive Shell): The absolute best out-of-the-box experience. Fish comes pre-configured with brilliant auto-suggestions based on your history, and full syntax highlighting (commands turn red if you spell them wrong, and green if they are valid) before you even hit enter.

If you install a new shell (e.g., sudo apt install zsh), the system doesn't automatically switch you over. You have to tell Linux to modify your specific user account to use it as the default. You do this with the chsh (Change Shell) command.

$ chsh -s $(which zsh)

# -s sets the shell. $(which zsh) automatically locates the exact directory where the Zsh binary is installed.


$ chsh -s $(which fish)

# If you prefer Fish, this command will permanently set it as your default shell.

⚠️ Final Step Required:
For the shell changes to take full effect across the system, you must completely log out of your user account and log back in (or simply reboot your computer). The next time you open the terminal window, you will be greeted by your brand new, high-performance environment.

5. Frequently Asked Questions

Will changing my shell break my Bash scripts?

No. Even if you use Zsh or Fish as your interactive daily driver, your automated scripts will still execute perfectly as long as you include the #!/bin/bash Shebang at the top of the file. The system will temporarily summon Bash just to run the script, then return you to your preferred shell.

Can I have different aliases for different users?

Absolutely. Because aliases are stored in the ~/.bashrc file located inside a specific user's /home directory, your custom shortcuts will not affect any other users on the system, nor will they affect the root administrator account.
