Basic Firewall: Closing the Gates

Your system has 65,535 doors. Stop leaving them wide open. | 👤 By JR Nation

Imagine your computer is a massive fortress with exactly 65,535 doors (called "Ports"). When you connect to the internet, applications use these doors to send and receive data. Your web browser uses Port 80, your email uses Port 25, and multiplayer games use their own custom doors.

A Firewall is the digital bouncer standing outside your fortress. Its job is to look at every single packet of data trying to enter or leave, check the rulebook, and decide whether to let it through or violently drop it. By default on a fresh Linux install, the bouncer is usually asleep. Let's wake him up.

1. Meet UFW (Uncomplicated Firewall)

In the past, configuring a Linux firewall meant writing incredibly complex rules using a low-level tool called iptables. It was a nightmare for beginners. Today, we use a frontend tool called UFW (Uncomplicated Firewall), which turns those complex rules into plain English.

If you are on Ubuntu, Mint, or Pop!_OS, it is likely already installed. If not, you can grab it using your package manager:

$ sudo apt install ufw

# Arch users: sudo pacman -S ufw

2. The Golden Rule: Deny All, Allow Out

Before we turn the firewall on, we need to establish the baseline rules. The standard security practice for any PC is simple: Nobody is allowed in unless I explicitly invited them, but I am allowed to leave whenever I want.

Run these two commands to set your default policies:

$ sudo ufw default deny incoming

$ sudo ufw default allow outgoing

This means if a random hacker runs an automated port scan against your IP address, your computer will simply ignore them. It won't even reply to say "I'm closed." It will act like it doesn't exist.

3. Opening Specific Doors

If you set your incoming policy to "deny," you will break things if you actually want to host a service. For example, if you want to control your PC remotely from your laptop using SSH, you need to open Port 22.

To open a door, you simply tell UFW to "allow" it:

$ sudo ufw allow ssh

# UFW is smart enough to know 'ssh' means Port 22.


$ sudo ufw allow 80/tcp

# Opens Port 80 (HTTP) specifically for standard web traffic.

⚠️ The Remote Server Death Trap:
If you are doing this on a remote VPS (like an AWS or DigitalOcean server) instead of the computer sitting on your desk, you MUST run sudo ufw allow ssh BEFORE you turn the firewall on. If you turn on a "deny all" firewall without opening the SSH port first, you will instantly sever your own connection and permanently lock yourself out of the server.

4. Throwing the Switch

Now that your baseline rules are set and your necessary doors are opened, it's time to bring the firewall online.

$ sudo ufw enable

Firewall is active and enabled on system startup

To check the status of your firewall and see exactly which doors are currently open and actively listening, use the status command:

$ sudo ufw status verbose

Your bouncer is now on duty. Unwanted traffic will be silently dropped before it ever touches your applications.
