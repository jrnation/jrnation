SSH: The Invisible Wire

Control your fortress from anywhere in the world. Securely. | 👤 By JR Nation

You don't need to be sitting in front of a keyboard and monitor to control a Linux machine. Whether it is a Raspberry Pi sitting in your closet, or a massive web server sitting in an Amazon data center in Tokyo, you can control it remotely using your terminal.

The tool that makes this possible is called SSH (Secure Shell). It creates a heavily encrypted, invisible wire between your laptop and the remote computer. Once connected, any command you type on your laptop is executed directly on the remote machine.

1. The Basic Connection

To connect to a remote computer, you need three things: the SSH client installed on your current machine, the SSH server (sshd) running on the remote machine, and the remote machine's IP address.

The command structure is simply ssh [username]@[IP_address].

$ ssh jaya@192.168.1.50

# Connects to the user 'jaya' on the computer located at that local IP address.

The first time you connect, your terminal will ask if you trust the "fingerprint" of the remote server. Type yes, enter the password for that user, and suddenly your terminal prompt will change. You are now inside the other computer.

2. The Danger of Passwords

By default, SSH uses standard typed passwords to verify who you are. This is a massive security risk. If you open Port 22 on your router to the open internet, automated hacker bots will find your IP address within minutes and start trying thousands of passwords a second (a "Brute Force" attack) to break in.

To fix this, true System Administrators do not use passwords for SSH. They use Cryptographic Keys.

3. Forging the Keys (ssh-keygen)

An SSH Key is actually a pair of files: a Public Key (the padlock) and a Private Key (the physical key). You put the padlock on the remote server, and you keep the physical key securely on your laptop. The padlock can *only* be opened by your specific file.

Step 1: Generate the Keys (On your current laptop)

$ ssh-keygen -t ed25519

# Generates a highly secure, modern key pair. Just press Enter to accept the default file locations.


Or use RSA, the older but widely compatible algorithm:

$ ssh-keygen -t rsa -b 4096

# Generates an RSA key pair with a length of 4096 bits.

Step 2: Send the Padlock to the Server

$ ssh-copy-id jaya@192.168.1.50

# This automatically copies your Public Key over to the remote server.

# You will need to enter the remote user's password one last time for this step.

Now, type ssh jaya@192.168.1.50 again. You will instantly log in without it ever asking for a password. The computers recognized the cryptographic handshake behind the scenes.

4. Locking the Door (Disable Passwords)

Now that your key is working, you need to tell the remote server to stop accepting typed passwords entirely. This makes you completely immune to brute-force dictionary attacks.

  1. SSH into the remote server.
  2. Edit the SSH configuration file using nano: sudo nano /etc/ssh/sshd_config
  3. Scroll down until you find the line that says PasswordAuthentication yes. (You may need to remove the # at the start of the line).
  4. Change the yes to a no.
  5. Save and exit nano.
  6. Restart the SSH service: sudo systemctl restart sshd
⚠️ DO NOT LOSE YOUR PRIVATE KEY. If you disabled password authentication and then your laptop breaks (losing your Private Key file), you are permanently locked out of the remote server. Always keep a backup of your ~/.ssh folder on a secure, offline USB drive.
